## セキュリティ概要

家計簿アプリケーションは、ユーザーの個人的な財務情報を扱うため、高いレベルのセキュリティが要求されます。本ドキュメントでは、アプリケーションのセキュリティ設計について詳述します。

## 脅威モデル

### 1. 想定される脅威

| 脅威                   | 影響度 | 対策優先度 | 説明                             |
| ---------------------- | ------ | ---------- | -------------------------------- |
| 不正ログイン           | 高     | 高         | 他人のアカウントへの不正アクセス |
| SQLインジェクション    | 高     | 高         | データベースへの不正アクセス     |
| XSS攻撃                | 中     | 高         | 悪意のあるスクリプト実行         |
| CSRF攻撃               | 中     | 中         | 意図しない操作の実行             |
| データ漏洩             | 高     | 高         | 個人情報の外部流出               |
| セッションハイジャック | 中     | 中         | セッションの乗っ取り             |

### 2. 資産の分類

| 資産             | 機密性 | 完全性 | 可用性 |
| ---------------- | ------ | ------ | ------ |
| ユーザー認証情報 | 極高   | 高     | 高     |
| 支出記録データ   | 高     | 高     | 中     |
| 予算設定データ   | 高     | 高     | 中     |
| システム設定     | 中     | 高     | 高     |

## 認証・認可設計

### 1. 認証方式

#### JWT（JSON Web Token）ベース認証
```java
@Configuration
@EnableWebSecurity
public class SecurityConfig {

    @Bean
    public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
        http
            .csrf().disable()
            .sessionManagement().sessionCreationPolicy(SessionCreationPolicy.STATELESS)
            .and()
            .authorizeHttpRequests(authz -> authz
                .requestMatchers("/api/v1/auth/**").permitAll()
                .requestMatchers("/api/v1/public/**").permitAll()
                .anyRequest().authenticated()
            )
            .oauth2ResourceServer().jwt();
        
        return http.build();
    }

    @Bean
    public PasswordEncoder passwordEncoder() {
        return new BCryptPasswordEncoder(12);
    }
}
```

### 1. バリデーション設計

#### Bean Validation使用

```java
public class CreateExpenseRequest {

    private LocalDate expenseDate;
}
```

#### カスタムバリデーター
```java
@Component
public class ExpenseValidator {
    
    public void validateExpenseCreation(CreateExpenseRequest request) {
        if (request.getExpenseDate().isAfter(LocalDate.now())) {
            throw new ValidationException("未来の日付は設定できません");
        }
        
        // SQLインジェクション対策
        if (containsSqlKeywords(request.getDescription())) {
            throw new ValidationException("不正な文字が含まれています");
        }
    }
    
    private boolean containsSqlKeywords(String input) {
        String[] sqlKeywords = {"DROP", "DELETE", "INSERT", "UPDATE", "SELECT"};
        String upperInput = input.toUpperCase();
        return Arrays.stream(sqlKeywords).anyMatch(upperInput::contains);
    }
}
```

## セッション管理

### 1. JWT トークン管理

#### トークンのライフサイクル
```java
@Service
public class JwtTokenService {
    
    private static final int ACCESS_TOKEN_VALIDITY = 24 * 60 * 60; // 24時間
    private static final int REFRESH_TOKEN_VALIDITY = 7 * 24 * 60 * 60; // 7日間
    
    public String generateAccessToken(UserDetails userDetails) {
        Map<String, Object> claims = new HashMap<>();
        claims.put("roles", userDetails.getAuthorities());
        return createToken(claims, userDetails.getUsername(), ACCESS_TOKEN_VALIDITY);
    }
    
    public String generateRefreshToken(String username) {
        return createToken(new HashMap<>(), username, REFRESH_TOKEN_VALIDITY);
    }
    
    @Async
    public void invalidateToken(String token) {
        // Redis等を使用してブラックリストに追加
        redisTemplate.opsForValue().set(
            "blacklist:" + token, 
            "true", 
            Duration.ofDays(7)
        );
    }
}
```

## データ保護

### 1. 暗号化

#### データベース暗号化
```java
@Entity
@Table(name = "user_accounts")
public class UserEntity {
    
    @Column(name = "email")
    @Convert(converter = EncryptionConverter.class)
    private String email;
    
    @Column(name = "password_hash")
    private String passwordHash; // BCryptでハッシュ化済み
}

@Converter
public class EncryptionConverter implements AttributeConverter<String, String> {
    
    @Autowired
    private EncryptionService encryptionService;
    
    @Override
    public String convertToDatabaseColumn(String attribute) {
        return encryptionService.encrypt(attribute);
    }
    
    @Override
    public String convertToEntityAttribute(String dbData) {
        return encryptionService.decrypt(dbData);
    }
}
```

#### パスワードハッシュ化
```java
@Service
public class PasswordService {
    
    private final PasswordEncoder passwordEncoder = new BCryptPasswordEncoder(12);
    
    public String hashPassword(String plainPassword) {
        return passwordEncoder.encode(plainPassword);
    }
    
    public boolean verifyPassword(String plainPassword, String hashedPassword) {
        return passwordEncoder.matches(plainPassword, hashedPassword);
    }
}
```

### 2. HTTPS強制

```java
@Configuration
public class HttpsConfig {
    
    @Bean
    public TomcatServletWebServerFactory servletContainer() {
        TomcatServletWebServerFactory tomcat = new TomcatServletWebServerFactory() {
            @Override
            protected void postProcessContext(Context context) {
                SecurityConstraint constraint = new SecurityConstraint();
                constraint.setUserConstraint("CONFIDENTIAL");
                SecurityCollection collection = new SecurityCollection();
                collection.addPattern("/*");
                constraint.addCollection(collection);
                context.addConstraint(constraint);
            }
        };
        tomcat.addAdditionalTomcatConnectors(redirectConnector());
        return tomcat;
    }
}
```

### 1. CSRF対策

```java
@Configuration
public class CsrfConfig {
    
    @Bean

        HttpSessionCsrfTokenRepository repository = new HttpSessionCsrfTokenRepository();
```

### 2. XSS対策

```java
@Component
public class XssFilterService {
    
    private final Policy policy;
    
    public XssFilterService() {
        this.policy = new PolicyFactory().allowElements("b", "i", "em", "strong");
    }
    
    public String sanitize(String input) {
        if (input == null) return null;
        return policy.sanitize(input);
    }
}

@ControllerAdvice
public class ResponseBodySanitizer implements ResponseBodyAdvice<Object> {
    
    @Override
    public Object beforeBodyWrite(Object body, 
                                 MethodParameter returnType,
                                 MediaType selectedContentType,
                                 Class<? extends HttpMessageConverter<?>> selectedConverterType,
                                 ServerHttpRequest request,
                                 ServerHttpResponse response) {
        // レスポンスデータのサニタイズ
        return sanitizeObject(body);
    }
}
```
