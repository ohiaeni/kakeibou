## システム概要

家計簿アプリは、ユーザーが日々の収支を記録・管理するためのWebアプリケーションです。Spring Bootを基盤としたマルチモジュール構成で構築されており、クリーンアーキテクチャの原則に従って設計されています。

**アーキテクチャパターン**: MPA（Multi-Page Application）
- サーバーサイドレンダリング（SSR）により、各ページを個別に配信
- Thymeleafテンプレートエンジンを使用したサーバーサイドテンプレート処理
- 従来のHTTPリクエスト/レスポンスサイクルによるページ遷移

```
│      Web        │    │ Application     │    │ Infrastructure  │
│   (Controller   │───▶│     Core        │───▶│   (Repository)  │

└─────┬───────────┘    └─────┬───────────┘    └─────┬───────────┘
      │                      │                      │
      │                      │                      │
      │                      ▼                      │
      │             ┌─────────────────┐             │
      └────────────▶│ System Common   │◀────────────┘
                    │   (Utilities)   │
                    └─────────────────┘
```

## モジュール構成

### 1. Web層 (web)
- **責務**: HTTPリクエストの受信、サーバーサイドレンダリング、レスポンスの返却
- **主要コンポーネント**:
  - MVC Controller（@Controller）
  - Thymeleafテンプレート
  - DTO (Data Transfer Object)
  - フォームバインディングクラス
  - 例外ハンドラー
  - セキュリティ設定
  - 静的リソース管理（CSS、JavaScript、画像等）

### 2. アプリケーション層 (application-core)
- **責務**: ビジネスロジックの実装
- **主要コンポーネント**:
  - Service クラス
  - ドメインモデル
  - ユースケース実装

### 3. インフラストラクチャ層 (infrastructure)
- **責務**: 外部システムとの連携（データベース、外部API等）
- **主要コンポーネント**:
  - Repository 実装
  - MyBatis Mapper
  - エンティティクラス
  - データベース設定

### 4. システム共通層 (system-common)
- **責務**: 全モジュールで共通利用される機能
- **主要コンポーネント**:
  - ユーティリティクラス
  - 共通例外クラス
  - 定数定義

## アーキテクチャ設計原則

### 1. レイヤード アーキテクチャ
- 各層は明確な責務を持ち、適切な抽象化レベルを維持します
- 上位層は下位層に依存しますが、下位層は上位層に依存しません

### 2. 依存関係の管理
- **Web層**: application-core および system-common に依存
- **Application Core層**: infrastructure および system-common に依存  
- **Infrastructure層**: system-common のみに依存
- **System Common層**: 他のモジュールに依存せず、すべての層から参照される基盤層

### 3. 共通機能の集約
- すべての層で必要となる共通機能（ユーティリティ、例外、定数等）は system-common に集約
- 重複コードを排除し、保守性を向上させます

## データベース設計原則

1. **UUID主キー**: 全テーブルでUUIDを主キーとして使用
2. **外部キー制約**: データ整合性を保証するため適切な外部キー制約を設定

## セキュリティ考慮事項

1. **パスワード**: BCryptによるハッシュ化
2. **SQL インジェクション対策**: MyBatisのパラメータバインディング使用
3. **CSRF対策**: Spring SecurityのCSRF保護機能を有効化

## パフォーマンス考慮事項

1. **データベース接続**: コネクションプールの適切な設定
2. **クエリ最適化**: 適切なインデックスの設定とN+1問題の回避
3. **キャッシュ**: Spring Cacheによる適切なキャッシュ戦略
4. **非同期処理**: 重い処理については非同期実行を検討
