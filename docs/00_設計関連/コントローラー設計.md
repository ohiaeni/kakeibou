## コントローラー概要

家計簿アプリのMVCコントローラー設計書です。サーバーサイドレンダリング（SSR）によるMPA（Multi-Page Application）アーキテクチャを採用し、Thymeleafテンプレートエンジンを使用してHTMLページを生成・配信します。

## 認証・セッション管理

### 認証方式
- **Spring Securityベースのフォーム認証**を使用
- セッションベース認証（HTTPセッション）
- ログイン成功時にセッションを作成
- セッション有効期限: 30分（アクティビティによる自動延長）

### セッション管理
- **セッション属性**:
  - ユーザーID
  - ユーザー名
  - ロール情報
- **セッション無効化**: ログアウト時、セッションタイムアウト時
- **CSRF保護**: Spring SecurityのCSRFトークンを使用

## コントローラー エンドポイント一覧

### 1. 認証関連 (Authentication)

#### ログイン画面表示
```http
GET /login
```
**説明**: ログインフォームを表示
**テンプレート**: `auth/login.html`
**レスポンス**: ログイン画面HTML

#### ログイン処理
```http
POST /login
```
**説明**: ユーザー認証処理
**パラメータ**:
- `username`: メールアドレス
- `password`: パスワード
- `_csrf`: CSRFトークン

**成功時**: ダッシュボードへリダイレクト (`/dashboard`)
**失敗時**: ログイン画面へリダイレクト（エラーメッセージ付き）

#### ユーザー登録画面表示
```http
GET /register
```
**説明**: ユーザー登録フォームを表示
**テンプレート**: `auth/register.html`
**レスポンス**: ユーザー登録画面HTML

#### ユーザー登録処理
```http
POST /register
```
**説明**: 新規ユーザー登録処理
**パラメータ**:
- `username`: ユーザー名
- `email`: メールアドレス
- `password`: パスワード
- `confirmPassword`: パスワード確認
- `_csrf`: CSRFトークン

**成功時**: ログイン画面へリダイレクト（成功メッセージ付き）
**失敗時**: 登録画面の再表示（バリデーションエラー付き）

#### ログアウト処理
```http
POST /logout
```
**説明**: ユーザーログアウト処理
**成功時**: ログイン画面へリダイレクト

### 2. ダッシュボード (Dashboard)

#### ダッシュボード画面表示
```http
GET /dashboard
```
**説明**: ダッシュボード画面を表示
**認証**: 必須

- カテゴリ別支出グラフ

### 3. 支出管理 (Expenses)

#### 支出記録画面表示
```http
GET /expenses/new
```
**説明**: 支出記録フォームを表示
**認証**: 必須
**テンプレート**: `expenses/form.html`
**レスポンス**: 支出記録画面HTML
**データ**: カテゴリ一覧

#### 支出記録処理
```http
POST /expenses
```
**説明**: 新規支出記録処理
**認証**: 必須
**パラメータ**:
- `amount`: 金額
- `categoryId`: カテゴリID
- `description`: 説明
- `expenseDate`: 支出日
- `_csrf`: CSRFトークン

**成功時**: 支出履歴画面へリダイレクト（成功メッセージ付き）
**失敗時**: 記録画面の再表示（バリデーションエラー付き）

#### 支出履歴画面表示
```http
GET /expenses
```
**説明**: 支出履歴一覧を表示
**認証**: 必須
**テンプレート**: `expenses/list.html`
**クエリパラメータ**:
- `page`: ページ番号（デフォルト: 1）
- `size`: 1ページあたりの件数（デフォルト: 20）
- `category`: カテゴリフィルター
- `startDate`: 開始日フィルター
- `endDate`: 終了日フィルター

**レスポンス**: 支出履歴画面HTML

#### 支出編集処理
```http
GET /expenses/{id}/edit
POST /expenses/{id}
```
**説明**: 支出記録の編集
**認証**: 必須
**GET**: 編集フォーム表示
**POST**: 更新処理

#### 支出削除処理
```http
POST /expenses/{id}/delete
```
**説明**: 支出記録の削除
**認証**: 必須
**成功時**: 支出履歴画面へリダイレクト

## エラーハンドリング

### エラーページ
- **400 Bad Request**: `error/400.html`
- **403 Forbidden**: `error/403.html`
- **404 Not Found**: `error/404.html`
- **500 Internal Server Error**: `error/500.html`

### バリデーションエラー表示
- フォームフィールドレベルでのエラー表示
- Spring Validation使用
- Thymeleafテンプレートでのエラーメッセージ表示

## セキュリティ考慮事項

1. **CSRF保護**: 全POSTリクエストにCSRFトークン必須
2. **セッション管理**: セッション固定攻撃対策
3. **認証・認可**: Spring Securityによる包括的な保護
4. **XSS対策**: Thymeleafによる自動エスケープ
5. **セキュリティヘッダー**: Spring Securityによるセキュリティヘッダー設定

## バリデーション規則

### 共通バリデーション
- **必須チェック**: `@NotBlank`, `@NotNull`
- **文字数制限**: `@Size`
- **形式チェック**: `@Email`, `@Pattern`

### フォーム別バリデーション

#### ユーザー登録・更新
- **ユーザー名**: 必須、3-50文字
- **メールアドレス**: 必須、メール形式、ユニーク制約
- **パスワード**: 必須、8文字以上、英数字含む

#### カテゴリ
- **カテゴリ名**: 必須、1-30文字

#### 支出記録
- **金額**: 必須、正の数値、最大999,999円
- **カテゴリID**: 必須、存在するカテゴリ
- **支出日**: 必須、過去または本日の日付

#### 予算
- **予算額**: 必須、正の数値、最大9,999,999円
- **年月**: 必須、有効な年月

## ページネーション

### リクエストパラメータ
- `page`: ページ番号（1から開始）
- `size`: 1ページあたりの件数
- `sort`: ソート条件

### テンプレート表示
- ページ番号リンク
- 前へ/次へボタン
- 総件数表示
- Thymeleafによるページネーション実装
