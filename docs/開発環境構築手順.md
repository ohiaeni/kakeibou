# 開発環境構築手順

## 前提条件

以下のソフトウェアがインストールされていることを確認してください：

- **Java**: OpenJDK 17以上
- **Git**: バージョン管理
- **IDE**: IntelliJ IDEA / Eclipse / Visual Studio Code
- **データベース**: PostgreSQL 14以上（本番環境用）

## 開発環境セットアップ

### 1. リポジトリのクローン

```bash
git clone https://github.com/ohiaeni/kakeibou.git
cd kakeibou
```

### 2. Gradle 設定確認

プロジェクトはGradle Wrapperを含んでいるため、Gradleを別途インストールする必要はありません。

```bash
# 権限設定（macOS/Linux）
chmod +x gradlew

# プロジェクトのビルド確認
./gradlew build
```

### 3. データベース設定

#### 開発環境（H2データベース）
デフォルトでH2インメモリデータベースを使用するため、特別な設定は不要です。

#### 本番環境（PostgreSQL）
PostgreSQLを使用する場合の設定：

```bash
# PostgreSQLのインストール（macOS）
brew install postgresql
brew services start postgresql

# データベースとユーザーの作成
createdb kakeibou_dev
psql -d kakeibou_dev -c "CREATE USER kakeibou_user WITH PASSWORD 'kakeibou_pass';"
psql -d kakeibou_dev -c "GRANT ALL PRIVILEGES ON DATABASE kakeibou_dev TO kakeibou_user;"
```

### 4. アプリケーション設定

#### 開発用設定ファイル作成
`backend/web/src/main/resources/application-dev.yml` を作成：

```yaml
spring:
  datasource:
    url: jdbc:postgresql://localhost:5432/kakeibou_dev
    username: kakeibou_user
    password: kakeibou_pass
    driver-class-name: org.postgresql.Driver
  
  jpa:
    hibernate:
      ddl-auto: validate
    show-sql: true
    properties:
      hibernate:
        format_sql: true
        dialect: org.hibernate.dialect.PostgreSQLDialect

logging:
  level:
    com.kakeibou: DEBUG
    org.springframework.security: DEBUG
```

### 5. IDE設定

#### IntelliJ IDEA
1. `File` -> `Open` でプロジェクトルートを選択
2. Gradleプロジェクトとして認識されることを確認
3. `File` -> `Settings` -> `Build, Execution, Deployment` -> `Build Tools` -> `Gradle`
   - `Use Gradle from`: `gradle-wrapper.properties file`
4. Lombokプラグインをインストール
5. `File` -> `Settings` -> `Build, Execution, Deployment` -> `Compiler` -> `Annotation Processors`
   - `Enable annotation processing` をチェック

#### Visual Studio Code
必要な拡張機能：
- Extension Pack for Java
- Spring Boot Extension Pack
- Lombok Annotations Support

## アプリケーション実行

### 1. 開発モードでの実行

```bash
# backend/web ディレクトリに移動
cd backend

# 開発プロファイルでアプリケーション起動
./gradlew :web:bootRun --args='--spring.profiles.active=dev'
```

### 2. アプリケーション確認

- **アプリケーション**: http://localhost:8080
- **API仕様書**: http://localhost:8080/swagger-ui.html
- **H2 コンソール**: http://localhost:8080/h2-console（開発環境のみ）

## テスト実行

### 1. 全テスト実行
```bash
./gradlew test
```

### 2. 特定モジュールのテスト実行
```bash
# webモジュールのテストのみ
./gradlew :web:test

# application-coreモジュールのテストのみ
./gradlew :application-core:test
```

### 3. テストカバレッジレポート生成
```bash
./gradlew jacocoTestReport
```
レポートは `build/reports/jacoco/test/html/index.html` で確認できます。

## コード品質チェック

### 1. 全チェック実行
```bash
./gradlew check
```

### 2. 個別チェック実行
```bash
# Checkstyle（コーディング規約チェック）
./gradlew checkstyleMain checkstyleTest

# SpotBugs（バグ検出）
./gradlew spotbugsMain spotbugsTest

# Spotless（コードフォーマット）
./gradlew spotlessCheck
```

### 3. コードフォーマット自動修正
```bash
./gradlew spotlessApply
```

```bash
# スキーマファイルの実行
psql -d kakeibou_dev -f backend/infrastructure/src/main/resources/schema.sql
```

### 2. テストデータ投入
```bash
# テストデータの投入
psql -d kakeibou_dev -f backend/infrastructure/src/main/resources/data.sql
```

## トラブルシューティング

### よくある問題と解決方法

#### 1. ビルドエラー
```bash
# キャッシュクリア
./gradlew clean

# 依存関係の再取得
./gradlew build --refresh-dependencies
```

#### 2. Lombokが動作しない
- IDEのLombokプラグインがインストールされているか確認
- annotation processing が有効になっているか確認

#### 3. データベース接続エラー
- PostgreSQLサービスが起動しているか確認
- データベース名、ユーザー名、パスワードが正しいか確認

#### 4. ポート衝突
```yaml
# application-dev.yml でポート変更
server:
  port: 8081
```

## 開発フロー

### 1. ブランチ戦略
- `main`: 本番リリースブランチ
- `develop`: 開発統合ブランチ
- `feature/*`: 機能開発ブランチ

### 2. コミット前チェック
```bash
# 必須チェック項目
./gradlew spotlessApply  # コードフォーマット
./gradlew test          # テスト実行
./gradlew check         # 品質チェック
```

### 3. プルリクエスト
- テストが全て通ることを確認
- コードレビューを受ける
- CI/CDパイプラインが成功することを確認
