# エラーハンドリングインタラクション

## インタラクション概要

支出登録画面で発生する各種エラーの処理と、ユーザーへの適切なフィードバック表示に関する動作を定義します。

## エラー分類とインタラクション

### 1. 入力バリデーションエラー

#### 金額入力エラー
```
[金額形式エラー]
    ├─ 数値以外の文字入力
    ├─ 負の値入力
    ├─ 小数点以下桁数超過
    ├─ 上限金額超過（100万円）
    └─ 下限金額未満（1円未満）

[金額エラー表示]
    ├─ フィールド枠線：赤色変更
    ├─ "正しい金額を入力してください"
    ├─ "1円以上100万円以下で入力してください"
    ├─ 入力値の自動補正提案
    └─ 通貨フォーマット例の表示

[自動補正機能]
    ├─ 全角数字→半角数字変換
    ├─ カンマ区切り自動挿入
    ├─ 小数点以下切り捨て
    ├─ 通貨記号自動除去
    └─ 空白文字除去
```

#### 日付入力エラー
```
[日付形式エラー]
    ├─ 存在しない日付
    ├─ 未来日付入力
    ├─ 過度に古い日付
    ├─ 形式不正
    └─ 必須項目未入力

[日付エラー処理]
    ├─ カレンダーピッカー表示
    ├─ "正しい日付を選択してください"
    ├─ 今日の日付への自動修正提案
    ├─ 月末日自動調整
    └─ うるう年対応チェック

[日付入力支援]
    ├─ 今日ボタン
    ├─ 昨日ボタン
    ├─ よく使う日付プリセット
    ├─ 相対日付入力（3日前等）
    └─ 音声日付入力
```

#### カテゴリー選択エラー
```
[カテゴリー未選択エラー]
    ├─ 必須項目チェック
    ├─ "カテゴリーを選択してください"
    ├─ カテゴリー一覧の強調表示
    ├─ よく使うカテゴリーの提案
    └─ 新規カテゴリー作成案内

[無効カテゴリーエラー]
    ├─ 削除済みカテゴリー検出
    ├─ 非公開カテゴリーアクセス
    ├─ "選択されたカテゴリーは無効です"
    ├─ 代替カテゴリーの提案
    └─ カテゴリー再選択の誘導
```

#### メモ・説明入力エラー
```
[文字数制限エラー]
    ├─ 最大文字数超過（500文字）
    ├─ リアルタイム文字数表示
    ├─ "あと○文字入力できます"
    ├─ 超過分の視覚的表示
    └─ 自動トリミング提案

[禁止文字エラー]
    ├─ HTMLタグ検出
    ├─ スクリプトコード検出
    ├─ 制御文字検出
    ├─ "使用できない文字が含まれています"
    └─ 自動エスケープ処理
```

### 2. 支払い方法エラー

#### クレジットカード情報エラー
```
[カード番号エラー]
    ├─ 桁数不正
    ├─ チェックディジット不正
    ├─ 存在しないカード会社
    ├─ 有効期限切れ
    └─ 無効なカード番号

[カードエラー表示]
    ├─ "正しいカード番号を入力してください"
    ├─ カード会社ロゴの非表示
    ├─ 入力形式ガイド表示
    ├─ 手動入力とスキャン選択
    └─ カスタマーサポート連絡先

[セキュリティエラー]
    ├─ CVV不正
    ├─ 有効期限不正
    ├─ "セキュリティコードを確認してください"
    ├─ カード裏面画像表示
    └─ セキュリティ説明
```

#### 電子マネー・QR決済エラー
```
[残高不足エラー]
    ├─ 利用可能残高確認
    ├─ "残高が不足しています"
    ├─ チャージ方法案内
    ├─ 他の支払い方法提案
    └─ 分割支払い提案

[連携エラー]
    ├─ API接続エラー
    ├─ 認証失敗
    ├─ "決済サービスに接続できません"
    ├─ 再認証案内
    └─ 手動入力への切り替え
```

### 3. データ保存エラー

#### ネットワークエラー
```
[接続エラー処理]
    ├─ オフライン状態検出
    ├─ "インターネット接続を確認してください"
    ├─ 自動保存機能
    ├─ オフライン作業継続
    └─ 接続復旧時の自動同期

[タイムアウトエラー]
    ├─ "保存処理がタイムアウトしました"
    ├─ 再試行ボタン表示
    ├─ 処理状況の説明
    ├─ データ損失防止処理
    └─ 自動リトライ機能

[サーバーエラー]
    ├─ 500番台エラー処理
    ├─ "サーバーで問題が発生しています"
    ├─ エラー詳細の非表示
    ├─ 技術サポート連絡先
    └─ 状況確認案内
```

#### データ整合性エラー
```
[重複データエラー]
    ├─ 同一取引検出
    ├─ "似たような支出が既に登録されています"
    ├─ 重複候補の表示
    ├─ 統合・分離選択
    └─ 継続登録オプション

[データ矛盾エラー]
    ├─ カテゴリー予算超過
    ├─ 日付範囲外
    ├─ "予算を超過しています。続行しますか？"
    ├─ 影響範囲の説明
    └─ 確認後の処理続行
```

### 4. 添付ファイルエラー

#### レシート画像エラー
```
[ファイル形式エラー]
    ├─ 対応外形式検出
    ├─ "JPEG、PNG形式のみ対応しています"
    ├─ ファイル変換サービス案内
    ├─ 代替アップロード方法
    └─ 手動入力への切り替え

[ファイルサイズエラー]
    ├─ 上限サイズ超過（10MB）
    ├─ "ファイルサイズが大きすぎます"
    ├─ 圧縮提案
    ├─ リサイズ機能提供
    └─ 複数ファイル分割提案

[画像認識エラー]
    ├─ OCR処理失敗
    ├─ "レシートを認識できませんでした"
    ├─ 手動入力案内
    ├─ 撮影方法改善提案
    └─ カメラ品質確認
```

#### アップロード処理エラー
```
[アップロード失敗]
    ├─ 通信エラー
    ├─ サーバー容量不足
    ├─ "ファイルのアップロードに失敗しました"
    ├─ 再試行ボタン
    └─ 後でアップロード機能

[ウイルススキャンエラー]
    ├─ 危険ファイル検出
    ├─ "安全性の確認ができませんでした"
    ├─ ファイル削除処理
    ├─ セキュリティ説明
    └─ 代替ファイル要求
```
