# 予算設定インタラクション

## インタラクション概要

ユーザーが月次予算を設定・調整・管理する際の画面動作を定義します。

## インタラクションフロー

### 1. 初期表示
```
[画面読み込み]
    ↓
[現在月の予算データ取得]
    ├─ 設定済み → 既存データ表示
    └─ 未設定 → デフォルト値設定
    ↓
[支出実績データ取得]
    ↓
[使用状況計算・表示更新]
    ├─ プログレスバー更新
    ├─ パーセンテージ計算
    └─ ステータス判定
```

### 2. 月次ナビゲーションインタラクション
```
[前月ボタンクリック]
    ↓
[月次変更可能性チェック]
    ├─ 過去12ヶ月以内 → 変更許可
    └─ 範囲外 → ボタン無効化
    ↓
[データ読み込み開始]
    ├─ ローディング表示
    ├─ 月次表示更新
    └─ 該当月データ取得

[次月ボタンクリック]
    ↓
[未来月制限チェック]
    ├─ 来月まで → 変更許可
    └─ 2ヶ月先以降 → ボタン無効化
    ↓
[新規月データ作成]
    ├─ 前月データをベースに初期化
    ├─ 自動配分提案表示
    └─ 未保存状態フラグ設定

[月次データ表示更新]
    ├─ フェードアウト → フェードイン（0.5s）
    ├─ カテゴリ別使用状況更新
    ├─ プログレスバー再描画
    └─ アラート状態再評価
```

### 3. 総予算設定インタラクション
```
[総予算フィールドフォーカス]
    ↓
[数値キーボード表示]（モバイル）
    ↓
[入力値監視]
    ├─ リアルタイム通貨フォーマット
    ├─ カンマ区切り表示
    └─ ¥記号自動付与

[バリデーション（リアルタイム）]
    ├─ 10,000未満 → "最低10,000円以上で設定してください"
    ├─ 1,000,000超過 → "1,000,000円以下で設定してください"
    ├─ 非数値 → "数値を入力してください"
    └─ 正常 → エラークリア・カテゴリ配分更新

[フォーカスアウト]
    ↓
[自動配分再計算]
    ├─ カテゴリ配分比率維持
    ├─ 端数調整処理
    └─ 配分結果反映

[総予算変更時の波及効果]
    ├─ カテゴリ予算自動調整
    ├─ 使用率再計算
    ├─ プログレスバー更新
    └─ アラート状態再評価
```

### 4. 自動配分インタラクション
```
[自動配分ボタンクリック]
    ↓
[配分計算モーダル表示]
    ├─ "過去3ヶ月の支出実績を分析中..."
    ├─ プログレスバー表示
    └─ 分析処理実行

[分析処理]
    ├─ 過去3ヶ月支出データ取得
    ├─ カテゴリ別平均算出
    ├─ 季節性ファクター適用
    ├─ 総予算に対する配分計算
    └─ 最低配分額保証処理

[配分結果プレビュー]
    ↓
[確認ダイアログ表示]
    ├─ 提案配分一覧表示
    ├─ 現在配分との比較表示
    ├─ 変更理由の説明
    └─ [適用][キャンセル]ボタン

[適用選択時]
    ├─ アニメーション付き値更新
    ├─ 各カテゴリ金額変更（0.5s）
    ├─ プログレスバー更新
    └─ モーダル閉じる

[配分ロジック詳細]
    ├─ 食費: 平均支出 × 1.1（余裕率）
    ├─ 固定費系: 平均支出 × 1.05
    ├─ 変動費系: 平均支出 × 1.2
    └─ 最低配分: 総予算の3%保証
```

### 5. カテゴリ別予算設定インタラクション
```
[カテゴリ予算フィールドフォーカス]
    ↓
[該当行ハイライト表示]
    ↓
[入力値監視]
    ├─ リアルタイム通貨フォーマット
    ├─ 合計予算計算
    └─ 残配分額表示

[合計チェック（リアルタイム）]
    ├─ 合計 > 総予算 → "総予算を超過しています"（赤色表示）
    ├─ 合計 < 総予算 × 0.8 → "配分不足です"（黄色警告）
    └─ 正常範囲 → エラークリア

[使用率計算・表示更新]
    ├─ 予算変更時即座に再計算
    ├─ プログレスバー更新（0.3s）
    ├─ ステータスアイコン更新
    └─ 色分け更新

[ステータス判定ロジック]
    ├─ 使用率 ≤ 70% → ✅（緑）
    ├─ 70% < 使用率 ≤ 90% → ⚠️（黄）
    ├─ 90% < 使用率 ≤ 100% → ⚠️（橙）
    └─ 使用率 > 100% → ❌（赤）

[カテゴリ間連動機能]
    ├─ 一つのカテゴリ増額時
    ├─ 他カテゴリの自動減額提案
    ├─ バランス調整ダイアログ表示
    └─ ユーザー選択による確定
```

### 6. アラート設定インタラクション
```
[アラート項目チェックボックス]
    ↓
[設定状態切り替え]
    ├─ チェック時 → 有効化・詳細設定表示
    └─ 未チェック時 → 無効化・設定非表示

[70%通知設定]
    ├─ 閾値: 70%固定
    ├─ 通知タイミング: 閾値到達時
    ├─ 通知方法選択: ブラウザ/アプリ内/メール
    └─ 通知頻度: 1日1回まで

[90%通知設定]
    ├─ 閾値: 90%固定
    ├─ 通知タイミング: 閾値到達時
    ├─ 通知方法選択
    └─ 通知頻度: 即座＋1日1回

[予算超過警告]
    ├─ 閾値: 100%
    ├─ 通知タイミング: 支出記録時
    ├─ 警告レベル: 高
    └─ 通知頻度: 毎回

[日次異常検知]
    ├─ 閾値: 平均日次支出の150%
    ├─ 学習期間: 過去30日
    ├─ 除外条件: 大型支出の手動除外
    └─ 通知頻度: 1日1回

[通知方法詳細設定]
    ├─ ブラウザ通知 → 権限要求
    ├─ アプリ内通知 → 常時有効
    ├─ メール通知 → メールアドレス確認
    └─ プッシュ通知 → PWA対応時
```

### 7. 保存処理インタラクション
```
[保存ボタンクリック]
    ↓
[最終バリデーション]
    ├─ 必須項目チェック
    ├─ 数値範囲チェック
    ├─ 合計整合性チェック
    └─ 論理的妥当性チェック

[バリデーション成功]
    ↓
[保存処理開始]
    ├─ ボタン無効化
    ├─ "保存中..." 表示
    ├─ スピナー表示
    └─ API送信

[API成功レスポンス]
    ├─ 成功メッセージ表示
    ├─ データ状態更新
    ├─ 変更フラグクリア
    └─ アラート設定反映

[API失敗レスポンス]
    ├─ エラーメッセージ表示
    ├─ 編集状態維持
    ├─ ボタン再有効化
    └─ リトライ提案

[バリデーション失敗]
    ├─ エラー箇所ハイライト
    ├─ エラーメッセージ表示
    ├─ 該当フィールドにフォーカス
    └─ 修正促進
```

### 8. リセット機能インタラクション
```
[リセットボタンクリック]
    ↓
[確認ダイアログ表示]
    "予算設定をリセットしますか？未保存の変更は失われます。"
    ├─ [キャンセル] → ダイアログ閉じる
    └─ [リセット] → リセット処理

[リセット処理]
    ├─ フォーム値を初期状態に復元
    ├─ 計算値リセット
    ├─ エラー状態クリア
    ├─ 変更フラグクリア
    └─ 表示更新

[自動配分リセット]
    ├─ 過去データに基づく配分
    ├─ アニメーション付き更新
    └─ 配分理由の表示
```

## 詳細動作仕様

### プログレスバー動作
```
[使用率計算]
    ├─ 使用額 ÷ 予算額 × 100
    ├─ 小数点以下1位まで表示
    └─ 100%超過時は100%表示（色で区別）

[色分けロジック]
    ├─ 0-50%: 緑色（#4CAF50）
    ├─ 51-70%: 薄緑色（#8BC34A）
    ├─ 71-85%: 黄色（#FF9800）
    ├─ 86-100%: 橙色（#FF5722）
    └─ 100%超過: 赤色（#F44336）

[アニメーション]
    ├─ 幅変更: 0.5s ease-out
    ├─ 色変更: 0.3s ease
    └─ 数値変更: 1.0s カウンターアニメーション
```

### 入力フィールド制御
```
[通貨フォーマット]
    ├─ 入力中: 数値のみ
    ├─ フォーカスアウト: ¥マーク付与
    ├─ カンマ区切り: 3桁毎
    └─ 小数点: 禁止（整数のみ）

[タブ順序制御]
    ├─ 総予算 → カテゴリ予算（上から順）
    ├─ Shift+Tab: 逆順移動
    └─ Enter: 次フィールドへ移動
```

## エラーハンドリング

### バリデーションエラー表示
```
[フィールドレベルエラー]
    ├─ 位置: 入力フィールド下部
    ├─ 色: 赤色（#F44336）
    ├─ アニメーション: スライドダウン 0.3s
    └─ 自動非表示: 正常値入力時

[フォームレベルエラー]
    ├─ 位置: 画面上部バナー
    ├─ スタイル: アラートバナー
    ├─ 自動非表示: 5秒後
    └─ 手動非表示: ×ボタン
```

### API通信エラー
```
[ネットワークエラー]
    ├─ オフライン → "オフラインです"
    ├─ タイムアウト → "通信タイムアウト"
    └─ サーバーエラー → "保存に失敗しました"

[業務エラー]
    ├─ 権限エラー → "この操作は許可されていません"
    ├─ データ競合 → "他のデバイスで変更されています"
    └─ 制約違反 → サーバーからのメッセージ表示
```

## アニメーション仕様

### 数値変更
- **カウンター**: 現在値 → 新値 1.0s ease-out
- **プログレスバー**: 幅変更 0.5s ease-out
- **色変更**: 0.3s ease

### UI フィードバック
- **フィールドフォーカス**: 枠線色変更 0.2s
- **ボタンホバー**: 背景色変更 0.2s
- **成功表示**: 緑色バナー スライドイン 0.5s
