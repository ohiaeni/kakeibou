# エラーハンドリングインタラクション

## インタラクション概要

ログイン画面で発生する各種エラーの処理と、ユーザーへの適切なフィードバック表示に関する動作を定義します。

## エラー分類とインタラクション

### 1. バリデーションエラー

#### メールアドレス形式エラー
```
[不正なメール形式入力]
    ↓
[リアルタイムエラー表示]
    ├─ フィールド枠線：赤色変更
    ├─ エラーアイコン表示（右側）
    ├─ エラーメッセージ：フィールド下部
    └─ "正しいメールアドレスを入力してください"

[エラー解消アニメーション]
    ├─ 正しい形式入力時
    ├─ 枠線色：赤→緑（0.3s）
    ├─ エラーメッセージ：フェードアウト
    └─ チェックアイコン表示
```

#### パスワード強度エラー
```
[弱いパスワード入力]
    ↓
[段階的警告表示]
    ├─ 8文字未満："8文字以上で入力してください"
    ├─ 英数字なし："英字と数字を含めてください"
    ├─ 一般的パスワード："より複雑なパスワードにしてください"
    └─ 強度インジケーター更新

[エラー状態の視覚表現]
    ├─ 警告アイコン（⚠️）
    ├─ 黄色枠線表示
    ├─ ヘルプテキスト表示
    └─ 改善提案表示
```

### 2. 認証エラー

#### ログイン失敗
```
[認証失敗レスポンス受信]
    ↓
[エラー分類処理]
    ├─ 401 Unauthorized → 認証情報不正
    ├─ 423 Locked → アカウントロック
    ├─ 429 Too Many Requests → 試行回数制限
    └─ 500 Server Error → サーバーエラー

[認証情報不正エラー]
    ├─ エラーバナー表示（画面上部）
    ├─ "メールアドレスまたはパスワードが正しくありません"
    ├─ パスワードフィールドクリア
    ├─ フォーカス：パスワード欄に移動
    └─ パスワードリセットリンク強調表示

[アカウントロックエラー]
    ├─ 重要度高エラー表示
    ├─ "アカウントがロックされています"
    ├─ ロック解除時刻表示
    ├─ サポート連絡先表示
    └─ パスワードリセット案内
```

#### 試行回数制限
```
[制限到達時の処理]
    ├─ ログインフォーム無効化
    ├─ カウントダウンタイマー表示
    ├─ "5回以上の失敗により一時的に制限されています"
    ├─ 残り時間表示："あと3分12秒で解除されます"
    └─ 代替手段（パスワードリセット）案内

[制限解除時の処理]
    ├─ フォーム再有効化
    ├─ "制限が解除されました"通知
    ├─ タイマー非表示
    └─ 通常状態復帰
```

### 3. ネットワークエラー

#### 接続エラー
```
[ネットワーク接続失敗]
    ↓
[接続状態判定]
    ├─ オフライン状態検知
    ├─ タイムアウト検知
    ├─ DNS解決失敗検知
    └─ プロキシエラー検知

[オフライン状態処理]
    ├─ オフラインバナー表示
    ├─ "インターネット接続を確認してください"
    ├─ ネットワーク状態アイコン表示
    ├─ 再接続ボタン提供
    └─ 接続復旧時自動リトライ

[タイムアウトエラー]
    ├─ "通信がタイムアウトしました"
    ├─ サーバー状況説明
    ├─ リトライボタン表示
    ├─ 手動リトライ機能
    └─ 状況改善まで待機案内
```

#### サーバーエラー
```
[5xx系エラー処理]
    ├─ 500 Internal Server Error
    ├─ 502 Bad Gateway
    ├─ 503 Service Unavailable
    └─ 504 Gateway Timeout

[サーバーエラー表示]
    ├─ "サーバーで問題が発生しています"
    ├─ "しばらくしてからお試しください"
    ├─ 状況確認リンク提供
    ├─ サポート連絡先表示
    └─ 自動リトライ機能（指数バックオフ）
```

### 4. セキュリティエラー

#### セッション期限切れ
```
[セッション無効検知]
    ├─ トークン期限切れ
    ├─ 不正なトークン
    ├─ セッション削除検知
    └─ セキュリティ違反検知

[セッション期限切れ処理]
    ├─ セキュリティ警告表示
    ├─ "セキュリティのため再ログインが必要です"
    ├─ 既存フォーム内容保持
    ├─ 再認証要求
    └─ ログイン成功後の状態復帰
```

#### 不正アクセス検知
```
[異常アクセスパターン検知]
    ├─ 短時間大量アクセス
    ├─ 異なる地域からの同時アクセス
    ├─ 不正なUser-Agent
    └─ ボット的な行動パターン

[セキュリティ警告処理]
    ├─ アクセス一時停止
    ├─ "セキュリティ確認が必要です"
    ├─ 追加認証要求（2FA等）
    ├─ メール通知送信
    └─ 管理者アラート発生
```

## エラー表示のデザインパターン

### バナー型エラー
```
[画面上部バナー]
    ├─ 背景色：エラーレベル別
    ├─ アイコン：エラー種別表示
    ├─ メッセージ：簡潔な説明
    ├─ 閉じるボタン（×）
    └─ 自動非表示（5秒後）

[色分けルール]
    ├─ 赤色：重大エラー（認証失敗等）
    ├─ 橙色：警告（制限等）
    ├─ 黄色：注意（形式エラー等）
    └─ 青色：情報（案内等）
```

### インライン型エラー
```
[フィールド直下エラー]
    ├─ 位置：該当入力欄下部
    ├─ アイコン：警告マーク
    ├─ テキスト：具体的な修正内容
    ├─ 色：赤色（#F44336）
    └─ アニメーション：スライドダウン

[フィールド状態変更]
    ├─ 枠線色変更
    ├─ 背景色の微調整
    ├─ アイコン表示
    └─ フォーカス状態の調整
```

### モーダル型エラー
```
[重要エラー用モーダル]
    ├─ 背景オーバーレイ
    ├─ 中央配置ダイアログ
    ├─ エラーアイコン（大）
    ├─ エラータイトル
    ├─ 詳細説明
    ├─ 対処方法案内
    └─ アクションボタン

[モーダル操作]
    ├─ 背景クリック無効
    ├─ Escapeキー無効
    ├─ 必須アクション要求
    └─ 適切な対処後のみ閉じる
```

## アニメーション仕様

### エラー表示アニメーション
- **バナー表示**: 上からスライドイン 0.5s ease-out
- **インラインエラー**: フェードイン + スライドダウン 0.3s
- **エラー解消**: フェードアウト 0.3s ease-in
- **シェイクエフェクト**: 重要エラー時 0.5s × 3回

### 状態変化アニメーション
- **フィールド状態**: 色変更 0.2s ease
- **アイコン変化**: 回転 + フェード 0.4s
- **プログレス表示**: 幅変更 リアルタイム
- **カウントダウン**: 数値変更 1.0s linear
