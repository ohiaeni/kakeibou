## 画面概要

ユーザーがアプリケーションにアクセスする際の認証画面です。メールアドレスとパスワードによる認証機能を提供します。

## 画面要素詳細

### 入力フィールド

1. **メールアドレス**
   - 形式: `<input type="email">`
   - バリデーション: メール形式チェック
   - 必須項目
   - プレースホルダー: "メールアドレスを入力"

2. **パスワード**
   - 形式: `<input type="password">`
   - バリデーション: 最低8文字
   - 必須項目
   - プレースホルダー: "パスワードを入力"
   - 表示/非表示切り替えアイコン

### コントロール要素

1. **ログイン状態を保持チェックボックス**
   - デフォルト: チェックなし
   - 機能: 次回自動ログイン

2. **ログインボタン**
   - 状態: 入力値有効時のみ有効化
   - スタイル: プライマリーボタン

3. **パスワードを忘れたリンク**
   - 遷移先: パスワードリセット画面

4. **新規登録リンク**
   - 遷移先: ユーザー登録画面

## バリデーション仕様

### リアルタイムバリデーション
- メールアドレス形式チェック
- パスワード文字数チェック
- 未入力チェック

### エラーメッセージ
- "正しいメールアドレスを入力してください"
- "パスワードは8文字以上で入力してください"
- "メールアドレスまたはパスワードが正しくありません"
- "ネットワークエラーが発生しました"

## セキュリティ仕様

1. **認証方式**: JWT (JSON Web Token)
2. **パスワード暗号化**: bcrypt
3. **ブルートフォース対策**: ログイン試行回数制限（5回）
4. **セッション管理**: トークンの有効期限設定（24時間）
