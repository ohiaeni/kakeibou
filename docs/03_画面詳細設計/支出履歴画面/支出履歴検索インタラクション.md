# 支出履歴検索インタラクション

## インタラクション概要

ユーザーが支出履歴を検索・フィルタリング・表示する際の画面動作を定義します。

## インタラクションフロー

### 1. 初期表示
```
[画面読み込み]
    ↓
[デフォルト検索条件設定]
    ├─ 期間: 今月（自動設定）
    ├─ カテゴリ: 全て
    ├─ 並び順: 日付順（新しい順）
    └─ 検索ワード: 空白
    ↓
[API検索実行]
    ↓
[ローディング表示] → スケルトン画面
    ↓
[結果表示]
    ├─ サマリー情報更新
    ├─ 一覧データ表示
    └─ ページネーション設定
```

### 2. 検索ワード入力インタラクション
```
[検索フィールドフォーカス]
    ↓
[プレースホルダー非表示]
    ↓
[文字入力開始]
    ↓
[デバウンス制御（300ms）]
    ├─ タイマーリセット
    └─ 新タイマー設定
    ↓
[タイマー満了時検索実行]
    ├─ 検索アイコン → スピナーに変更
    ├─ API検索リクエスト
    └─ 結果反映

[検索結果表示]
    ├─ ヒット件数表示
    ├─ 該当なし → "検索条件に一致する記録がありません"
    ├─ 検索ワードハイライト表示
    └─ 検索履歴保存（最大10件）
```

### 3. 期間選択インタラクション
```
[期間選択ボタンクリック]
    ↓
[ドロップダウン展開]
    ├─ 今日
    ├─ 昨日
    ├─ 今週
    ├─ 今月 ← (デフォルト選択)
    ├─ 先月
    ├─ 過去3ヶ月
    ├─ 過去6ヶ月
    ├─ 過去1年
    └─ カスタム期間...

[プリセット期間選択]
    ├─ 即座に期間確定
    ├─ ドロップダウン閉じる
    └─ 自動検索実行

[カスタム期間選択]
    ↓
[日付ピッカーモーダル表示]
    ├─ 開始日カレンダー
    ├─ 終了日カレンダー
    ├─ プリセットボタン群
    └─ [適用][キャンセル]ボタン

[日付バリデーション]
    ├─ 開始日 > 終了日 → "開始日は終了日より前の日付を選択してください"
    ├─ 期間 > 1年 → "検索期間は1年以内で指定してください"
    ├─ 未来日付 → "未来の日付は選択できません"
    └─ 正常 → 期間確定・検索実行
```

### 4. カテゴリフィルターインタラクション
```
[カテゴリボタンクリック]
    ↓
[マルチセレクトドロップダウン展開]
    ├─ ☑️ 全て (デフォルト選択)
    ├─ ☐ 🍽️ 食費
    ├─ ☐ 🚃 交通費
    ├─ ☐ 🎮 娯楽費
    ├─ ☐ 🧴 日用品
    ├─ ☐ 💊 医療費
    ├─ ☐ 📚 教育費
    └─ ☐ 📦 その他

[個別カテゴリ選択]
    ├─ "全て"のチェック自動解除
    ├─ 選択カテゴリのチェック切り替え
    ├─ 選択数カウンター更新
    └─ リアルタイム検索実行

["全て"選択]
    ├─ 他の全選択解除
    ├─ "全て"のみ選択状態
    └─ 全カテゴリ表示

[選択状態表示]
    ├─ 選択なし → "カテゴリを選択"
    ├─ 1つ選択 → カテゴリ名表示
    ├─ 複数選択 → "○件選択中"
    └─ 全て選択 → "全て"
```

### 5. 並び順変更インタラクション
```
[並び順ドロップダウンクリック]
    ↓
[選択肢表示]
    ├─ 📅 日付順（新しい順） ← デフォルト
    ├─ 📅 日付順（古い順）
    ├─ 💰 金額順（高い順）
    ├─ 💰 金額順（安い順）
    └─ 🏷️ カテゴリ順

[並び順選択]
    ├─ 選択項目ハイライト
    ├─ ドロップダウン閉じる
    ├─ ローディング表示
    └─ データ再取得・表示更新

[ソートアニメーション]
    ├─ 既存リストフェードアウト（0.2s）
    ├─ ローディングスピナー表示
    ├─ 新リストデータ取得
    └─ 新リストフェードイン（0.3s）
```

### 6. 一覧表示インタラクション

#### 無限スクロール（モバイル）
```
[スクロール位置監視]
    ↓
[下端近接検知（残り20%）]
    ↓
[追加データ読み込み判定]
    ├─ 次ページあり → 追加読み込み
    └─ 最終ページ → "全ての記録を表示しました"

[追加読み込み]
    ├─ ローディングインジケーター表示
    ├─ API追加データ取得
    ├─ 既存リストに追加（アニメーション）
    └─ ローディング非表示
```

#### ページネーション（デスクトップ）
```
[ページ番号クリック]
    ↓
[ページ変更確認]
    ├─ 同一ページ → 処理なし
    └─ 異なるページ → ページ読み込み

[ページ読み込み]
    ├─ ローディングオーバーレイ表示
    ├─ API指定ページデータ取得
    ├─ リスト内容置換
    ├─ ページネーション更新
    └─ 画面トップへスクロール

[前へ/次へボタン]
    ├─ 無効状態チェック
    ├─ 有効時: ページ移動実行
    └─ 無効時: クリック無効
```

### 7. 検索結果表示インタラクション

#### サマリー情報更新
```
[検索結果取得完了]
    ↓
[カウンターアニメーション]
    ├─ 総金額: 現在値 → 新値（1秒間）
    ├─ 件数: 現在値 → 新値（0.8秒間）
    └─ 期間表示: 即座に更新

[予算比較表示]
    ├─ 月次予算データ取得
    ├─ 使用率計算
    ├─ プログレスバー更新
    └─ 残額/超過額表示
```

#### 空の検索結果
```
[検索結果0件]
    ↓
[空状態表示]
    ├─ 📊 アイコン表示
    ├─ "検索条件に一致する記録がありません"
    ├─ 検索条件見直し提案
    └─ [条件リセット]ボタン

[条件リセットボタン]
    ├─ 全フィルター初期化
    ├─ 検索ワードクリア
    ├─ 期間を今月に設定
    └─ 自動再検索
```

### 8. リアルタイム検索最適化

#### 検索状態管理
```
[検索条件変更検知]
    ↓
[検索状態更新]
    ├─ URL更新（ブラウザ履歴対応）
    ├─ 検索条件永続化
    └─ API リクエスト準備

[API呼び出し制御]
    ├─ 進行中リクエストキャンセル
    ├─ 新リクエスト送信
    ├─ レスポンス順序保証
    └─ エラーハンドリング
```

#### パフォーマンス最適化
```
[レスポンス高速化]
    ├─ 結果キャッシュ（5分間）
    ├─ 部分マッチング
    ├─ インデックス活用
    └─ 段階的データ読み込み

[UI レスポンス向上]
    ├─ デバウンス制御
    ├─ スケルトン画面
    ├─ プリロード機能
    └─ エラー時リトライ
```

## エラーハンドリング

### 検索エラー
```
[API エラーレスポンス]
    ├─ 400 Bad Request → "検索条件が正しくありません"
    ├─ 404 Not Found → "データが見つかりません"
    ├─ 500 Server Error → "検索に失敗しました。しばらくしてからお試しください"
    └─ タイムアウト → "検索がタイムアウトしました。条件を見直してください"

[ネットワークエラー]
    ├─ オフライン → "オフラインです。接続を確認してください"
    ├─ 接続不安定 → "接続が不安定です。しばらくしてからお試しください"
    └─ [リトライ]ボタン表示
```

### バリデーションエラー
```
[入力値エラー]
    ├─ 期間指定エラー → フィールド下にエラー表示
    ├─ 検索ワード長すぎ → "50文字以下で入力してください"
    └─ 不正文字 → "使用できない文字が含まれています"

[エラー解消]
    ├─ 入力修正時エラークリア
    ├─ 正常値確認後再検索
    └─ エラー状態リセット
```

## アニメーション仕様

### 検索関連
- **検索実行**: 検索アイコン回転 → スピナー 0.3s
- **結果更新**: 既存リスト フェードアウト → 新リスト フェードイン 0.5s
- **フィルター適用**: 該当行ハイライト → 非該当行フェードアウト 0.4s

### カウンター
- **数値変更**: イージング関数使用 1.0s
- **金額表示**: 桁区切りアニメーション 0.8s
- **進捗バー**: 幅変更アニメーション 1.2s
