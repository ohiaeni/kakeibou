# ユーザー登録インタラクション

## インタラクション概要

新規ユーザーがアカウント情報を入力して登録を行う際の画面動作を定義します。

## インタラクションフロー

### 1. 初期表示
```
[画面読み込み]
    ↓
[フォーム初期化]
    ↓
[入力フィールドフォーカス設定] → ユーザー名欄にフォーカス
```

### 2. 入力操作シーケンス

#### ユーザー名入力
```
[ユーザー名入力開始]
    ↓
[文字数カウンター表示] → "0/20文字"
    ↓
[リアルタイムバリデーション]
    ├─ 2文字未満 → "2文字以上入力してください"
    ├─ 20文字超過 → "20文字以下で入力してください"
    ├─ 無効文字 → "使用できない文字が含まれています"
    └─ 有効 → 重複チェック実行

[重複チェック（デバウンス: 500ms）]
    ├─ 利用可能 → 緑色チェックアイコン表示
    └─ 重複 → "このユーザー名は既に使用されています"
```

#### メールアドレス入力
```
[メールアドレス入力]
    ↓
[メール形式チェック]
    ├─ 無効 → "正しいメールアドレスを入力してください"
    └─ 有効 → 重複チェック実行

[重複チェック（デバウンス: 1秒）]
    ├─ 利用可能 → 緑色チェックアイコン表示
    └─ 重複 → "このメールアドレスは既に登録されています"
```

#### パスワード入力
```
[パスワード入力]
    ↓
[パスワード強度チェック]
    ├─ 弱 → 赤色バー・"パスワード強度: 弱"
    ├─ 中 → 黄色バー・"パスワード強度: 中"
    └─ 強 → 緑色バー・"パスワード強度: 強"

[文字種別チェック]
    ├─ 英字のみ → "数字を含めてください"
    ├─ 数字のみ → "英字を含めてください"
    └─ 英数字含む → "✓ 英数字を含んでいます"

[文字数チェック]
    ├─ 8文字未満 → "8文字以上で入力してください"
    └─ 8文字以上 → "✓ 文字数要件を満たしています"
```

#### パスワード確認入力
```
[パスワード確認入力]
    ↓
[一致チェック（リアルタイム）]
    ├─ 不一致 → "パスワードが一致しません"
    └─ 一致 → "✓ パスワードが一致しています"
```

### 3. 利用規約同意
```
[利用規約リンククリック]
    ↓
[モーダルウィンドウ表示] → 利用規約テキスト表示
    ↓
[閉じるボタン] または [背景クリック] → モーダル閉じる

[同意チェックボックス]
    ├─ チェック → アカウント作成ボタン有効化チェック
    └─ 未チェック → アカウント作成ボタン無効状態維持
```

### 4. アカウント作成実行
```
[アカウント作成ボタンクリック]
    ↓
[最終バリデーション]
    ├─ エラーあり → 該当フィールドにエラー表示
    └─ 正常 → 登録処理開始

[ローディング開始]
    ├─ ボタンテキスト変更: "作成中..."
    ├─ ボタン無効化
    ├─ スピナー表示
    └─ フォーム入力無効化

[API登録リクエスト]
    ├─ 成功
    │   ├─ [成功メッセージ表示] → "アカウントが作成されました"
    │   ├─ [確認メール送信通知] → "確認メールを送信しました"
    │   └─ [ログイン画面へリダイレクト（3秒後）]
    │
    └─ 失敗
        ├─ [エラーメッセージ表示]
        ├─ [ローディング終了]
        └─ [フォーム再有効化]
```

## 詳細動作仕様

### パスワード強度インジケーター

#### 表示要素
- **プログレスバー**: 0-100%の強度表示
- **色**: 弱(赤) / 中(黄) / 強(緑)
- **テキスト**: 強度レベル表示
- **チェックリスト**: 
  - ✓ 8文字以上
  - ✓ 英字を含む
  - ✓ 数字を含む
  - ✓ 特殊文字を含む（推奨）

#### 強度判定ロジック
```
[基本点数: 0点]
├─ 8文字以上: +25点
├─ 英字含む: +25点
├─ 数字含む: +25点
├─ 特殊文字含む: +15点
└─ 12文字以上: +10点

[判定]
├─ 0-40点: 弱
├─ 41-70点: 中
└─ 71点以上: 強
```

### リアルタイムバリデーション

#### デバウンス制御
- **ユーザー名重複チェック**: 500ms
- **メール重複チェック**: 1000ms
- **入力検証**: 300ms

#### API呼び出し制御
```
[入力イベント]
    ↓
[既存タイマークリア]
    ↓
[新タイマー設定]
    ↓
[タイマー満了時API実行]
    ├─ ローディングインジケーター表示
    ├─ API呼び出し
    └─ 結果反映・ローディング非表示
```

### フォーム状態管理

#### アカウント作成ボタン有効化条件
```
[有効化チェック]
├─ ユーザー名: 有効 & 重複なし
├─ メールアドレス: 有効 & 重複なし
├─ パスワード: 8文字以上 & 英数字含む
├─ パスワード確認: 一致
└─ 利用規約: 同意

[全条件満足] → ボタン有効化
[条件不足] → ボタン無効状態維持
```

## エラーハンドリング

### サーバーエラー分類
```
[HTTPステータス別処理]
├─ 400 Bad Request
│   ├─ バリデーションエラー → 各フィールドにエラー表示
│   └─ 一般的な不正リクエスト → "入力内容に問題があります"
│
├─ 409 Conflict
│   └─ 重複エラー → "このデータは既に使用されています"
│
├─ 422 Unprocessable Entity
│   └─ 業務ロジックエラー → サーバーからのメッセージを表示
│
├─ 429 Too Many Requests
│   └─ レート制限 → "しばらくしてからお試しください"
│
└─ 500 Internal Server Error
    └─ サーバーエラー → "サーバーエラーが発生しました"
```

### エラー表示パターン
```
[フィールドエラー]
    ├─ 位置: 入力フィールド下部
    ├─ 色: 赤色（#F44336）
    ├─ アニメーション: フェードイン
    └─ 自動非表示: エラー解消時

[グローバルエラー]
    ├─ 位置: 画面上部
    ├─ スタイル: アラートバナー
    ├─ 自動非表示: 5秒後
    └─ 手動非表示: ×ボタンクリック
```

## アニメーション仕様

### 入力フィードバック
- **フォーカス**: 枠線色変更 0.2s ease
- **バリデーション成功**: チェックアイコン フェードイン 0.3s
- **エラー表示**: エラーメッセージ スライドダウン 0.3s
- **強度バー**: プログレス アニメーション 0.5s ease-out

### 画面遷移
- **成功時**: 成功メッセージ → 3秒後フェードアウト → ページ遷移
- **エラー時**: エラーメッセージ シェイク アニメーション 0.5s
