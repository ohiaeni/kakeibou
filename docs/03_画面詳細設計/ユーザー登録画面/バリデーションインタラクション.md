# バリデーションインタラクション

## インタラクション概要

ユーザー登録画面における各入力項目のリアルタイムバリデーション処理と、ユーザーへの適切なフィードバック表示を定義します。

## バリデーション対象項目

### 1. メールアドレス検証

#### 形式チェック
```
[入力値監視]
    ├─ onInput イベント（リアルタイム）
    ├─ onChange イベント（フォーカス移動時）
    ├─ onBlur イベント（フィールド離脱時）
    └─ onSubmit イベント（送信時最終確認）

[形式チェックロジック]
    ├─ 空文字チェック
    ├─ RFC 5322準拠正規表現
    ├─ 最大長制限（254文字）
    ├─ 日本語ドメイン対応
    └─ 特殊文字制限

[視覚的フィードバック]
    ├─ 入力中：枠線色変化なし
    ├─ 不正形式：赤枠 + エラーアイコン
    ├─ 正常形式：緑枠 + チェックアイコン
    └─ 空文字：デフォルト状態維持
```

#### 重複チェック（非同期）
```
[重複チェック実行タイミング]
    ├─ 形式チェック通過後
    ├─ 入力停止から500ms後（デバウンス）
    ├─ フィールド離脱時
    └─ フォーム送信前

[API呼び出し処理]
    ├─ Loading状態表示
    ├─ スピナーアイコン表示
    ├─ 同時リクエスト制御
    ├─ タイムアウト処理（5秒）
    └─ エラーハンドリング

[結果フィードバック]
    ├─ 利用可能："✓ 利用可能です"
    ├─ 重複あり："! このメールアドレスは既に登録されています"
    ├─ 確認エラー："サーバーエラーが発生しました。再度お試しください"
    └─ タイムアウト："確認がタイムアウトしました。後でもう一度お試しください"
```

### 2. パスワード強度チェック

#### 強度判定アルゴリム
```
[段階的強度判定]
    ├─ レベル1：8文字以上
    ├─ レベル2：大文字小文字含む
    ├─ レベル3：数字含む
    ├─ レベル4：記号含む
    └─ レベル5：16文字以上

[一般的パスワード検証]
    ├─ 辞書アタック対策
    ├─ 一般的パスワードリスト照合
    ├─ 順次文字列検出（123456、abcdef等）
    ├─ 繰り返し文字検出（111111、aaaaaa等）
    └─ 個人情報含有チェック（名前、メール等）

[リアルタイム強度表示]
    ├─ プログレスバー（0-100%）
    ├─ 色段階表示（赤→黄→緑）
    ├─ テキスト表示（弱い→普通→強い）
    ├─ 改善提案表示
    └─ 必要文字種別案内
```

#### パスワード一致確認
```
[確認パスワード検証]
    ├─ リアルタイム一致確認
    ├─ 文字単位での比較
    ├─ 不一致時即座に警告
    ├─ 一致時確認表示
    └─ 空白時状態リセット

[一致状態の視覚表現]
    ├─ 不一致：赤枠 + "! パスワードが一致しません"
    ├─ 一致：緑枠 + "✓ パスワードが一致しています"
    ├─ 未入力：デフォルト状態
    └─ 入力中：判定保留状態
```

### 3. 氏名バリデーション

#### 文字種チェック
```
[許可文字種]
    ├─ ひらがな（ぁ-ゖ）
    ├─ カタカナ（ァ-ヶー）
    ├─ 漢字（常用漢字 + 人名用漢字）
    ├─ 英字（A-Z, a-z）
    ├─ 半角スペース（区切り文字）
    └─ 一部記号（ハイフン等）

[文字数制限]
    ├─ 姓：1-20文字
    ├─ 名：1-20文字
    ├─ 全角文字優先
    ├─ 半角変換提案
    └─ 文字数カウンター表示

[入力支援機能]
    ├─ 全角自動変換
    ├─ ふりがな自動入力提案
    ├─ 読み方確認機能
    └─ 入力形式案内
```

#### ふりがなバリデーション
```
[ふりがな形式チェック]
    ├─ ひらがなのみ許可
    ├─ 漢字名との対応確認
    ├─ 文字数上限（姓名各30文字）
    ├─ 禁止文字チェック
    └─ 読み方妥当性確認

[自動入力支援]
    ├─ IME連携によるふりがな取得
    ├─ 漢字変換履歴からの推測
    ├─ 一般的な読み方データベース照合
    ├─ 手動修正機能
    └─ 読み方確認ダイアログ
```

### 4. 生年月日バリデーション

#### 日付形式チェック
```
[入力形式対応]
    ├─ プルダウン選択方式
    ├─ 年：1900-現在年
    ├─ 月：1-12
    ├─ 日：月に応じた適切な日数
    └─ うるう年対応

[妥当性チェック]
    ├─ 存在しない日付検出
    ├─ 未来日付制限
    ├─ 年齢上下限チェック（13-120歳）
    ├─ 論理的矛盾検出
    └─ 形式統一確認

[年齢計算と表示]
    ├─ リアルタイム年齢計算
    ├─ 年齢確認表示
    ├─ 利用規約年齢制限確認
    ├─ 未成年者向け案内
    └─ 年齢非表示オプション
```

### 5. 利用規約同意チェック

#### 必須項目確認
```
[同意状態管理]
    ├─ 利用規約同意チェック
    ├─ プライバシーポリシー同意
    ├─ メール配信同意（任意）
    ├─ データ利用同意
    └─ 第三者提供同意（必要時）

[同意確認インタラクション]
    ├─ チェックボックス状態管理
    ├─ 規約リンク動作確認
    ├─ 別ウィンドウ/モーダル表示
    ├─ スクロール状態確認
    └─ 同意内容理解確認
```

## バリデーション実行タイミング

### リアルタイムバリデーション
```
[トリガーイベント]
    ├─ onInput：文字入力毎
    ├─ onChange：値変更時
    ├─ onBlur：フォーカス離脱時
    ├─ onFocus：フォーカス取得時
    └─ カスタムイベント：API結果等

[デバウンス制御]
    ├─ 入力中：バリデーション遅延
    ├─ 停止検知：500ms後実行
    ├─ 重複処理防止
    ├─ パフォーマンス最適化
    └─ ユーザビリティ向上
```

### フォーム送信時バリデーション
```
[最終確認処理]
    ├─ 全項目再バリデーション
    ├─ エラー項目の特定
    ├─ 最初のエラー項目へフォーカス
    ├─ エラーサマリー表示
    └─ 送信ボタン制御

[エラー状態の統合表示]
    ├─ エラー件数表示
    ├─ エラー項目一覧
    ├─ 修正必要箇所へのリンク
    ├─ 一括修正支援
    └─ 進捗状況表示
```

## エラーメッセージ設計

### メッセージトーン
```
[メッセージ指針]
    ├─ 丁寧で親しみやすい表現
    ├─ 具体的な修正方法の提示
    ├─ 否定的表現の回避
    ├─ 建設的な提案の提供
    └─ ユーザーを責めない表現

[段階的メッセージ]
    ├─ 注意レベル："～をご確認ください"
    ├─ 警告レベル："～を修正してください"
    ├─ エラーレベル："～が入力されていません"
    ├─ 成功レベル："✓ 入力内容を確認しました"
    └─ 情報レベル："～をお勧めします"
```

### コンテキスト別メッセージ
```
[項目別カスタマイズ]
    ├─ メールアドレス：
    │   ├─ "正しいメールアドレスを入力してください"
    │   ├─ "このメールアドレスは既に登録されています"
    │   └─ "確認のため再度入力してください"
    ├─ パスワード：
    │   ├─ "8文字以上で入力してください"
    │   ├─ "英字と数字を組み合わせてください"
    │   └─ "より安全なパスワードにしてください"
    └─ 氏名：
        ├─ "姓名をそれぞれ入力してください"
        ├─ "ひらがなで入力してください"
        └─ "正しい文字で入力してください"
```
