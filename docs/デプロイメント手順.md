# デプロイメント手順

## デプロイメント概要

家計簿アプリケーションのデプロイメントは以下の環境を想定しています：

- **開発環境**: ローカル開発
- **ステージング環境**: 本番前テスト
- **本番環境**: ユーザー向けサービス

## ビルドとパッケージング

### 1. アプリケーションビルド

```bash
# プロダクションビルド
./gradlew clean build -Pprofile=production

# JARファイルの確認
ls -la backend/web/build/libs/
# 出力例: web-0.0.1-SNAPSHOT.jar
```

### 2. Docker イメージ作成

#### Dockerfile（backend/Dockerfile）
```dockerfile
FROM openjdk:17-jre-slim

# アプリケーション用ユーザー作成
RUN adduser --system --group appuser

# 作業ディレクトリ設定
WORKDIR /app

# JARファイルをコピー
COPY web/build/libs/*.jar app.jar

# ユーザー権限変更
RUN chown appuser:appuser app.jar
USER appuser

# ヘルスチェック設定
HEALTHCHECK --interval=30s --timeout=3s --start-period=60s --retries=3 \
  CMD curl -f http://localhost:8080/actuator/health || exit 1

# アプリケーション起動
EXPOSE 8080
ENTRYPOINT ["java", "-jar", "app.jar"]
```

#### Docker イメージビルド
```bash
# Dockerイメージ作成
docker build -t kakeibou-backend:latest backend/

# イメージ確認
docker images | grep kakeibou
```

## 環境別デプロイメント設定

### 1. 開発環境（Development）

#### docker-compose.yml
```yaml
version: '3.8'
services:
  app:
    build: ./backend
    ports:
      - "8080:8080"
    environment:
      - SPRING_PROFILES_ACTIVE=dev
      - SPRING_DATASOURCE_URL=jdbc:h2:mem:devdb
    depends_on:
      - postgres
    volumes:
      - ./logs:/app/logs

  postgres:
    image: postgres:14
    environment:
      POSTGRES_DB: kakeibou_dev
      POSTGRES_USER: kakeibou_user
      POSTGRES_PASSWORD: kakeibou_pass
    ports:
      - "5432:5432"
    volumes:
      - postgres_data:/var/lib/postgresql/data

volumes:
  postgres_data:
```

#### 起動コマンド
```bash
# 開発環境起動
docker-compose up -d

# ログ確認
docker-compose logs -f app
```

### 2. ステージング環境（Staging）

#### Kubernetes設定例
```yaml
# k8s/staging/namespace.yml
apiVersion: v1
kind: Namespace
metadata:
  name: kakeibou-staging

---
# k8s/staging/deployment.yml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: kakeibou-backend
  namespace: kakeibou-staging
spec:
  replicas: 2
  selector:
    matchLabels:
      app: kakeibou-backend
  template:
    metadata:
      labels:
        app: kakeibou-backend
    spec:
      containers:
      - name: app
        image: kakeibou-backend:staging
        ports:
        - containerPort: 8080
        env:
        - name: SPRING_PROFILES_ACTIVE
          value: "staging"
        - name: DATABASE_URL
          valueFrom:
            secretKeyRef:
              name: db-secrets
              key: url
        resources:
          requests:
            memory: "512Mi"
            cpu: "250m"
          limits:
            memory: "1Gi"
            cpu: "500m"
        livenessProbe:
          httpGet:
            path: /actuator/health
            port: 8080
          initialDelaySeconds: 60
          periodSeconds: 30
        readinessProbe:
          httpGet:
            path: /actuator/health/readiness
            port: 8080
          initialDelaySeconds: 30
          periodSeconds: 10

---
# k8s/staging/service.yml
apiVersion: v1
kind: Service
metadata:
  name: kakeibou-service
  namespace: kakeibou-staging
spec:
  selector:
    app: kakeibou-backend
  ports:
  - port: 80
    targetPort: 8080
  type: LoadBalancer
```

## CI/CD パイプライン

### 1. GitHub Actions設定

#### .github/workflows/deploy.yml
```yaml
name: Deploy to Production

on:
  push:
    branches: [main]

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - uses: actions/setup-java@v3
        with:
          java-version: '17'
          distribution: 'temurin'
      
      - name: Cache Gradle packages
        uses: actions/cache@v3
        with:
          path: |
            ~/.gradle/caches
            ~/.gradle/wrapper
          key: ${{ runner.os }}-gradle-${{ hashFiles('**/*.gradle*', '**/gradle-wrapper.properties') }}
      
      - name: Run tests
        run: ./gradlew test jacocoTestReport
      
      - name: Upload coverage to Codecov
        uses: codecov/codecov-action@v3

  build-and-deploy:
    needs: test
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      
      - name: Set up JDK 17
        uses: actions/setup-java@v3
        with:
          java-version: '17'
          distribution: 'temurin'
      
      - name: Build application
        run: ./gradlew build -Pprofile=production
      
      - name: Build Docker image
        run: |
          docker build -t ${{ secrets.DOCKER_REGISTRY }}/kakeibou-backend:${{ github.sha }} backend/
          docker tag ${{ secrets.DOCKER_REGISTRY }}/kakeibou-backend:${{ github.sha }} ${{ secrets.DOCKER_REGISTRY }}/kakeibou-backend:latest
      
      - name: Push to registry
        run: |
          echo ${{ secrets.DOCKER_PASSWORD }} | docker login -u ${{ secrets.DOCKER_USERNAME }} --password-stdin ${{ secrets.DOCKER_REGISTRY }}
          docker push ${{ secrets.DOCKER_REGISTRY }}/kakeibou-backend:${{ github.sha }}
          docker push ${{ secrets.DOCKER_REGISTRY }}/kakeibou-backend:latest
      
      - name: Deploy to production
        run: |
          # Kubernetesデプロイメント更新
          kubectl set image deployment/kakeibou-backend app=${{ secrets.DOCKER_REGISTRY }}/kakeibou-backend:${{ github.sha }} -n kakeibou-production
          kubectl rollout status deployment/kakeibou-backend -n kakeibou-production
```

## データベースマイグレーション

### 1. マイグレーション戦略

#### Flyway設定
```gradle
// build.gradle
plugins {
    id 'org.flywaydb.flyway' version '8.5.13'
}

flyway {
    url = 'jdbc:postgresql://localhost:5432/kakeibou_prod'
    user = 'kakeibou_user'
    password = 'kakeibou_pass'
    locations = ['classpath:db/migration']
}
```

#### マイグレーションファイル例
```sql
-- src/main/resources/db/migration/V1__Initial_schema.sql
CREATE TABLE user_accounts (
    id UUID PRIMARY KEY NOT NULL,
    username VARCHAR(128) NOT NULL,
    email VARCHAR(256) NOT NULL UNIQUE,
    password_hash VARCHAR(256) NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

### 2. マイグレーション実行

```bash
# 本番環境でのマイグレーション実行
./gradlew flywayMigrate -Pflyway.url=jdbc:postgresql://prod-db:5432/kakeibou_prod
```

## モニタリングとログ

### 1. ログ設定

#### Logback設定（logback-spring.xml）
```xml
<configuration>
    <springProfile name="production">
        <appender name="FILE" class="ch.qos.logback.core.rolling.RollingFileAppender">
            <file>/app/logs/application.log</file>
            <rollingPolicy class="ch.qos.logback.core.rolling.TimeBasedRollingPolicy">
                <fileNamePattern>/app/logs/application.%d{yyyy-MM-dd}.%i.gz</fileNamePattern>
                <maxFileSize>100MB</maxFileSize>
                <maxHistory>30</maxHistory>
                <totalSizeCap>3GB</totalSizeCap>
            </rollingPolicy>
            <encoder>
                <pattern>%d{yyyy-MM-dd HH:mm:ss.SSS} [%thread] %-5level %logger{36} - %msg%n</pattern>
            </encoder>
        </appender>
        
        <root level="INFO">
            <appender-ref ref="FILE" />
        </root>
    </springProfile>
</configuration>
```

## バックアップとリストア

### 1. データベースバックアップ

```bash
# 日次バックアップスクリプト
#!/bin/bash
DATE=$(date +%Y%m%d_%H%M%S)
pg_dump -h prod-db -U kakeibou_user kakeibou_prod > backup_${DATE}.sql
gzip backup_${DATE}.sql
aws s3 cp backup_${DATE}.sql.gz s3://kakeibou-backups/
```

### 2. リストア手順

```bash
# バックアップからのリストア
aws s3 cp s3://kakeibou-backups/backup_20240115_120000.sql.gz .
gunzip backup_20240115_120000.sql.gz
psql -h prod-db -U kakeibou_user -d kakeibou_prod < backup_20240115_120000.sql
```
